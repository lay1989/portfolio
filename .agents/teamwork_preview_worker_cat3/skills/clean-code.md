# Clean Code Skill

This skill embodies the principles of "Clean Code" by Robert C. Martin (Uncle Bob). Use it to transform "code that works" into "code that is clean."

## 🧠 Core Philosophy
> "Code is clean if it can be read, and enhanced by a developer other than its original author." — Grady Booch

## Key Guidelines
1. **Meaningful Names**: Intention-revealing, searchable names. Nouns for classes, verbs for methods.
2. **Functions**: Small, do one thing, one level of abstraction, minimal arguments, no side effects.
3. **Comments**: Don't comment bad code - rewrite it. Express intent in code itself.
4. **Formatting**: Newspaper metaphor, declare variables close to usage, vertical density.
5. **Error Handling**: Use exceptions over return codes, catch-finally, do not return/pass null.
6. **Classes**: Small, single responsibility principle (SRP).
