# Project: Homepage Refactoring and CRO Improvement

This project aims to refactor the homepage (`content/index.html`) to eliminate AI slop patterns and improve conversion rate optimization (CRO) based on `.agentrules`.

## Architecture
The application is a custom static site generator (SSG) system:
- **Build Input**:
  - `content/index.html`: Contains the body fragment of the homepage.
  - Other page fragments in `content/` and `components/` (e.g., header, footer).
  - Templates in `templates/` (e.g., base layout wrapper).
- **Build Output**:
  - `index.html` (and other root-level HTML files): Compiled outputs generated by `scripts/build-html.js`.
  - `tailwind.css`: Compiled utility classes output by Tailwind CLI from `style.css`.
  - `bundle.js`: Bundled and minified client-side JavaScript built by `esbuild` from `script.js`.
- **Flow**:
  1. Tailwind CSS compiler compiles `style.css` into `tailwind.css`.
  2. esbuild bundles `script.js` into `bundle.js`.
  3. Static Site Generator (SSG) compiles files in `content/` using Cheerio and templates in `templates/` to produce root-level HTML files.

## Code Layout
- `content/index.html` - Homepage layout and body content (main source file for this task).
- `style.css` - Tailwind input styles and custom animation rules.
- `tailwind.config.js` - Tailwind configuration.
- `package.json` - Build scripts (`npm run build`).
- `scripts/build-html.js` - Static site builder script.
- `templates/` - Layout HTML files.
- `components/` - Common UI fragment templates.
