# Color Contrast & Visibility Analysis Report

**Date**: July 14, 2026  
**Challenger Role**: Contrast & Visibility Challenger (Milestone 3, Gen 2)  
**Scope**: Empirically verify computed colors and contrast of Lucide icons on `projects.html` and blog pages (`blog.html` and all `blog-*.html` files) in light and dark modes, ensuring compliance with WCAG AA standards (contrast >= 4.5:1).

---

## Executive Summary

1. **Clean Audit for Primary Pages**: The main pages (`projects.html`, `blog.html`) and individual blog post pages (`blog-*.html`) have **0 color contrast violations**. All Lucide icons on these pages use high-contrast color classes (such as `text-foreground`, `text-primary dark:text-accent`, and `text-muted-foreground`) resolving to contrast ratios >= 4.5:1 in both light and dark modes.
2. **Critical Contrast Violations in Project Detail Case Studies**: On individual project detail pages (`projects/*.html`), there are **187 contrast violations** across 7 files. These are caused by the templating script `scripts/build-html.js` using `text-accent` (`#FF6B35`) directly on light cream backgrounds (`#F5F0EA` or `#EDE8E1`) for section headers and list items, resulting in poor contrast ratios of **2.50:1** and **2.33:1** (failing the WCAG AA 4.5:1 requirement).
3. **Critical HTML Validation Bug (Self-Closing `<i>` Tags)**: Across `blog.html` and all `blog-*.html` pages, Lucide icons are written with self-closing syntax (e.g. `<i data-lucide="..." />`). HTML5 does not support self-closing tags for inline elements, causing browsers and parser libraries to nest subsequently rendered content deeply inside the unclosed `<i>` elements.

## 2. Accent Color Contrast Violations on Cream Backgrounds

We specifically verified that no `.lucide` icons use classes that render as `#FF6B35` (accent color) on `#F5F0EA` or `#EDE8E1` (cream backgrounds) in light mode, unless there is a high-contrast override class.

In `blog.html` and `blog-*.html`, there are **no instances** of this violation. All accent colors on these pages in light mode are wrapped in high-contrast classes (like `text-primary`).

However, in **all generated project detail case studies** (`projects/*.html`), there are **187 occurrences** of this exact violation.

### Contrast Breakdown of Accent Color (#FF6B35) on Cream Backgrounds

1. **Accent vs Page Background (`#F5F0EA`)**:
   - Accent Luminance: `0.3230`
   - Page Background Luminance: `0.8747`
   - **Contrast Ratio**: **2.50:1** (Fails WCAG AA 4.5:1)
2. **Accent vs Card Background (`#EDE8E1`)**:
   - Accent Luminance: `0.3230`
   - Card Background Luminance: `0.8059`
   - **Contrast Ratio**: **2.33:1** (Fails WCAG AA 4.5:1)
3. **Accent vs Light Accent Overlay Background (`#EFDCD0` / `bg-accent/10`)**:
   - Accent Luminance: `0.3230`
   - Overlay Background Luminance: `0.7410`
   - **Contrast Ratio**: **2.14:1** (Fails WCAG AA 4.5:1)

### Location of Violations in `projects/*.html` Case Studies

## 3. Structural Bug: Self-Closing `<i>` Tags

In `content/blog.html` and `content/blog-*.html` (all blog posts), icons are coded as:
`... <i data-lucide="..." class="..." aria-hidden="true"/> ...`

### Impact
Standard HTML5 parsers do not support self-closing syntax for inline elements (`<i>`). The slash `/` is ignored. Browsers treat this as an unclosed opening tag, nesting subsequent content and elements inside it.
- **Cheerio Parsing Impact**: Cheerio nests footer tags and subsequent icons within the unclosed `i` tags, resulting in invalid DOM tree paths.
- **Verification Correction**: For this analysis, our verification script applied an in-memory regular expression replacement (`rawContent.replace(/<i([^>]*)\/>/g, '<i$1></i>')`) to restore standard HTML5 structure before calculating contrast ratios. This reduced the parsed icon count from an artificial **1,908** to the correct **524** icons.

## 4. Suggested Mitigations (Actionable Recommendations)

1. **Fix `scripts/build-html.js` Icons**:
   Update the class list for icons in the case study helper functions within `scripts/build-html.js` to use `text-primary dark:text-accent` instead of plain `text-accent`.
   - *Example change*:
     ```javascript
     // Change from:
     <i data-lucide="workflow" class="w-8 h-8 text-accent" aria-hidden="true"></i>
     // To:
     <i data-lucide="workflow" class="w-8 h-8 text-primary dark:text-accent" aria-hidden="true"></i>
     ```
2. **Standardize Self-Closing Tags**:
   Search and replace all self-closing `<i>` elements in `content/` with standard closing tags:
   - Find: `/><\/i>| \/>|\/>`
   - Replace with: `><\/i>` or `></i>`
